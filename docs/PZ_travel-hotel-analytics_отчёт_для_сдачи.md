## 2. Где проект и почему нет ссылки на Replit

| Вопрос | Ответ |
|--------|--------|
| **Где актуальная версия?** | **GitHub** — https://github.com/igrgor/AIKIVAVIORA-Travel-MCP-Workspace (ветка `main`, коммит с watchlist, отчётами, Weather MCP) |
| **Была ли версия на Replit?** | Да. Replit Agent собрал **UI Shell MVP** (Hotel Analytics Pro). Проект выгружен в архив `Hotel-Insight-Hub.zip` и приложен к сдаче. |
| **Почему нет URL `.replit.app`?** | Публичный деплой Replit для этой части ДЗ **не продлевался**: лимит Replit Agent/API был израсходован на часть 1 (крипто-MCP); часть 2 (отели) дорабатывалась **локально в Cursor**, а не через новый Publish на Replit. |
| **Как проверить работу?** | Клонировать репозиторий с GitHub → `pnpm install` → api-server `:3001` + hotel-analytics `:5173`. Скриншоты в отчёте сняты с **локального запуска**. |
| **Чем заменяем ссылку Replit для сдачи?** | **GitHub** (исходники + история) + **архив Replit** (исходный UI Shell) + **скриншоты** + **docx-отчёт**. |

> **Итог для проверяющего:** часть 1 ДЗ (крипта) сдана с Replit URL; часть 2 (отели) — с **GitHub** как основной точкой доступа, потому что актуальный код после доработок опубликован там, а не на Replit.

## 3. Реализованный функционал

### 3.1. Базовый MVP (Replit — UI Shell)

- трёхколоночный интерфейс в стиле Bloomberg Terminal;
- русскоязычный UI, тёмная тема AIKIVAVIORA;
- KPI-карточки, карточки отелей, матрица сравнения;
- журнал активности, источники данных, пульс рынка;
- исследовательский чат (демо-диалог).

### 3.2. Доработки в Cursor (фазы 1–3)

| Модуль | Реализация |
|--------|------------|
| **Мой список** | Watchlist в localStorage, вкладка «Мои», кнопка в статус-баре |
| **Сравнение** | До 9 отелей, единое состояние, панель активных сравнений, снимок |
| **Отчёты** | Сохранение из чата + снимок сравнения, экспорт Markdown |
| **Погода MCP** | Weather Panel: сейчас / 7 дней / почасово, график 30 дней |
| **Каталог** | 24 демо-отеля; целевой STR 847 — вкладка «Все (847)» с пометкой «в разработке» |
| **Прозрачность UX** | Бейджи Демо / В разработке / Заглушка / Локально — не выглядит как баг |
| **API stubs** | `GET/PUT /api/workspace`, `GET/POST /api/reports` |
| **DB stubs** | Drizzle schema `workspace`, `reports` (без миграции) |

### 3.3. Weather MCP (задел под ДЗ-2)

| Эндпоинт | MCP tool |
|----------|----------|
| `GET /api/weather/current` | `get_current_conditions` |
| `GET /api/weather/daily` | `get_forecast` (daily) |
| `GET /api/weather/hourly` | `get_forecast` (hourly) |

Документация: `docs/WEATHER_MCP.md`.
